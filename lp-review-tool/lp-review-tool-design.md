# Web デザインレビューツール 設計ドキュメント

## 1. 要件

### 背景

実装エージェント（CC）がFigmaからHTMLを生成するワークフローにおいて、デザイナーによるビジュアルレビューとフィードバックの仕組みが必要。LP、HP、その他Webページ全般が対象。

### 要件

| # | 要件 |
|---|------|
| R1 | PC向け・スマホ向け両方のWebデザインをレビューできる |
| R2 | デザイナーはPCやスマホなど実機で確認・フィードバックできる |
| R3 | 実装側はフィードバックを既存の仕組み（GitHub PR）で確認・対応できる |
| R4 | デザイナーはステークホルダー（クライアント等）をレビュアーとして追加できる |
| R5 | 個人情報の取り扱いを最小限にする（アカウント管理は外部サービスに委ねる） |
| R6 | 公開前のデザインを扱うため、デザインが漏洩しないようセキュリティを担保する |
| R7 | 無償で利用できる |

### 設計方針

| 対応要件 | 方針 | 判断理由 |
|----------|------|----------|
| R1 | iframe幅の切り替えでDesktop / Tablet / Mobileを表示 | レスポンシブ対応のデザインをビューポート別に確認できる |
| R2 | デザイナーはブラウザだけで完結する | コードやGitHubに触れさせない。学習コストゼロ |
| R2 | プレビューURLを実機ブラウザでも開ける | 認証済みなら同じURLでスマホ実機から確認・フィードバック可能 |
| R3 | フィードバックをGitHub PRコメントに集約する | 独自DBに保存しない。CCの既存ワークフロー（PR通知→修正→push）をそのまま使える |
| R3 | PRコメントからレビューツールへリンクする | `?focus=fb-003` で該当箇所を展開表示。PR↔ツール間をシームレスに行き来できる |
| R4 | メンバー招待機能を設ける | メールアドレスで招待し、Google認証でログイン。GitHubアカウント不要 |
| R5 | 認証をGoogle OAuthに委ねる | パスワードや個人情報を自前で持たない。D1にはメール・名前・アバターURLのみ |
| R6 | Google認証 + プロジェクト単位のアクセス制御 | 認証済みかつプロジェクトメンバーのみがプレビューにアクセスできる |
| R7 | Cloudflare無料枠 + GitHub API無料枠で構成する | Workers（10万req/日）、D1（5GB）、GitHub API（5,000req/時）すべて無料枠内 |

### 技術選定

| 領域 | 選定 |
|------|------|
| フロントエンド | SolidJS |
| バックエンド | Hono on Cloudflare Workers |
| DB | Cloudflare D1 |
| 認証 | Google OAuth 2.0 |
| フィードバック管理 | GitHub API（PRコメント） |

## 2. ユーザー体験

> 概念図: `ux.mermaid`

### 登場人物

| 誰 | 役割 |
|-----|------|
| デザイナー（オーナー） | プロジェクト作成、メンバー招待、プレビュー確認、フィードバック、承認 |
| CC（実装エージェント） | HTML生成、PR作成、フィードバック対応 |

### 補足

- **デザイナー**: オーナーを兼ねる。初回にプロジェクト作成・メンバー招待を行い、以降はプレビュー確認・フィードバック・承認を担当する。触るのはブラウザだけ
- **CC**: 触るのはGitHub PRだけ。フィードバックはPRコメントとして届くので既存ワークフローと変わらない
- **レビューツール**: デザイナーとGitHub PRの橋渡し役。デザイナーのビジュアル操作をPRコメントに変換し、PRコメントをピンとして画面上に表示する
- **フォーカス機能**: PRコメントにはレビューツールへのリンク（`?focus=fb-003`）が含まれる。PRからリンクを踏むと該当コメントが展開した状態で表示され、他のピンはマーカーのみで閉じている
- **レビューサイクル**: コメント → 修正 → 再確認を繰り返し、全て解決したらデザイナーが「承認」→ PRにApproveが投稿される

## 3. アーキテクチャ

> 概念図: `architecture.mermaid`

### 構成要素

| 要素 | 責務 |
|------|------|
| SolidJS SPA | レビュー画面（iframe + canvas + コメントUI） |
| Hono API | 認証、アクセス制御、GitHub APIプロキシ |
| Cloudflare D1 | ユーザー・プロジェクト・メンバー管理 |
| GitHub API | フィードバック（PRコメント）の保存・取得・承認 |
| プレビュー配信 | PRブランチのHTMLを静的配信 |

### 補足

- **Cloudflare Workers**: すべてのサーバーサイドコンポーネントが載る。外部に立てるサーバーはない
- **SolidJS SPA**: iframe（HTMLプレビュー）+ canvas（ピン・アノテーション描画層）+ コメントUIで構成。iframeの中のHTMLは汚さない
- **Hono API**: Google OAuthの処理、D1でのアクセス権確認、GitHub APIへのプロキシを担う。デザイナーはGitHubアカウント不要（APIがトークンを代理使用）
- **D1**: 3テーブルのみ（users, projects, project_members）。フィードバックデータは保存しない。セッションは暗号化Cookieで管理
- **GitHub**: フィードバックの実体はPRコメント。承認はPR Reviewとして投稿。リポジトリのHTMLソースがプレビューの元になる
- **静的配信**: PRブランチのHTMLをWorkersから配信。pushされるたびに更新される
- **スクショは撮らない**: PRコメントのリンクからレビューツールに飛べば実物が見える。R2等のストレージは不使用
- **コスト $0**: Workers無料枠（10万req/日）、D1無料枠（5GB）、GitHub API無料枠（5,000req/時）ですべて収まる
